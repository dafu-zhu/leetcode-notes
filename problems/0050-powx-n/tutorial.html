Pow(x, n)

Medium leetcode.com ↗

Approach: binary exponentiation, written recursively.

Intuition

Computing x^n by multiplying x by itself n times works, but it is very slow when n is large. Here n can be about two billion, which is far too many multiplications.

Binary exponentiation is much faster. It rests on a single relationship between a power and a smaller power:

The recurrence If n is even:  x^n = (x^2)^(n/2)
If n is odd:   x^n = x × (x^2)^((n-1)/2)

Both cases square the base and cut the exponent roughly in half. Since every step at least halves n, the number of multiplications drops from n to about log₂ n. For n near two billion, that is roughly 31 steps instead of two billion.

Negative powers are handled on their own. Compute x to the power |n| first, then take the reciprocal if n was negative, because x^(-n) = 1 / x^n.

Watch it run

Take x = 2, n = 10. The recursion follows the solid arrows down, squaring the base and halving the exponent, until it reaches the base case. Then it returns up the dashed arrows, multiplying by the base only at the levels where n was odd.

Algorithm

  1. Define a recursive function helper(x, n):
    • If x is 0, return 0. Zero to any positive power is zero.
    • If n is 0, return 1. Any number to the power zero is one. This is the base case that stops the recursion.
    • Otherwise compute res = helper(x * x, n // 2), which squares the base and halves the exponent.
    • If n is odd, return x * res. If n is even, return res.
  2. Call helper(x, abs(n)) so the exponent is never negative during the recursion.
  3. If n was negative, return 1 / result. Otherwise return result.

The full solution

class Solution:
    def myPow(self, x: float, n: int) -> float:
        def helper(x, n):
            if x == 0:                     # 0 to any positive power is 0
                return 0
            if n == 0:                     # base case: anything to the power 0 is 1
                return 1
            res = helper(x * x, n // 2)    # square the base, halve the exponent
            return x * res if n % 2 else res   # odd n keeps one extra factor of x

        res = helper(x, abs(n))            # compute the magnitude first
        return res if n >= 0 else 1 / res  # reciprocal when n was negative

Time is O(log n), since the exponent is halved on every call. Space is O(log n) for the recursion stack, which reaches a depth of about 31 at the largest allowed n. This solution ships with a 500 case fuzz suite in test_solution.py.

Takeaway

Pattern to carry forward

When the input is an exponent, do not multiply n times. Square the base and halve the exponent on each step. This turns O(n) work into O(log n). An odd exponent simply keeps one extra factor of the current base.

The same idea powers modular exponentiation (pow(a, b, m)), fast matrix power (which computes Fibonacci in O(log n)), and binary lifting (jumping 2^k ancestors in a tree). Learn the shape once and it comes back often.
